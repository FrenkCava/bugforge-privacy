Installazione

BugForge al momento non è pubblicata sul Chrome Web Store: l'installazione avviene tramite link diretto ricevuto dal team o dal referente che ti ha fornito l'estensione.

Apri il link ricevuto
Apri il link diretto che ti è stato fornito per installare BugForge.
Aggiungi l'estensione a Chrome
Segui la pagina di installazione e conferma l'aggiunta dell'estensione quando Chrome lo richiede.
Aggiungi BugForge alla barra
Clicca il puzzle 🧩 in alto a destra, poi la puntina 📌 accanto a BugForge per mantenerlo sempre visibile.
Vai sulla pagina da testare
Apri il sito o la pagina che vuoi analizzare, poi clicca l'icona 🐞 nella barra di Chrome.
ℹ️
Come funziona la finestra: BugForge si apre in una finestra separata affiancata al browser. La finestra originale si riduce alla metà sinistra dello schermo, BugForge occupa la metà destra. Puoi continuare a navigare normalmente sulla pagina sorgente.
Cosa fa

Funzionalità

Tutto in un'unica interfaccia: cattura, annota, classifica ed esporta.

📸
Screenshot schermo intero
Cattura l'intera finestra sorgente. Ogni cattura diventa una tab separata, navigabile e eliminabile.
✂️
Cattura + Ritaglia
Cattura e attiva la modalità crop: trascina per definire l'area da conservare, poi conferma.
✏️
Annotazioni canvas
Penna a mano libera, frecce, rettangoli e testo. 5 colori, spessore regolabile, Undo per ogni passo.
🏷️
Classificazione strutturata
Tipo BUG o CHANGE REQUEST, ID auto-generato (BUG-01, MOD-01…), Severità Alta/Media/Bassa.
🎤
Dettatura vocale (sperimentale)
Funzione sperimentale con microfono indipendente per "Problema" e "Soluzione Proposta". Il comportamento può variare in base a permessi, browser e disponibilità del riconoscimento vocale.
📋
Console Logs
Raccoglie log, warn, error, exception e unhandledrejection dalla pagina sorgente.
🌐
Network Monitor
Intercetta fetch e XHR: metodo, URL (senza parametri sensibili), status code e durata ms.
📄
Export HTML + PNG
Report HTML completo con tutti i dati, oppure PNG dello screenshot corrente con annotazioni.
Tutorial

Come si usa

Segui questi 6 passi per creare una segnalazione completa.

Passo 1 di 6
Apri BugForge

Naviga sulla pagina che presenta il problema e clicca l'icona nella barra Chrome. BugForge si apre in una finestra separata affiancata.

  • 1Clicca l'icona nella barra degli strumenti di Chrome
  • 2La finestra originale si sposta a sinistra, BugForge si apre a destra
  • 3In alto vedi il badge con titolo e URL della pagina sorgente
  • ✓Console logs e chiamate di rete vengono raccolti automaticamente dall'apertura in poi
💡
Finestra già aperta? Cliccando l'icona di nuovo Chrome porta in primo piano la finestra esistente senza aprirne un'altra.

Passo 3 di 6
Ritaglia (opzionale)

Se hai usato "Cattura + Ritaglia", la modalità crop si attiva automaticamente dopo la cattura.

  • 1Clicca e trascina sul canvas per selezionare l'area da conservare
  • 2L'area selezionata rimane visibile, il resto si scurisce con overlay
  • 3Clicca ✔ Conferma Ritaglio per applicare il ritaglio
  • 4Oppure ✖ Annulla per tornare allo screenshot integro
⚠️
Durante il ritaglio tutti gli strumenti di annotazione sono disabilitati. Applica prima il ritaglio, poi annota.

Passo 4 di 6
Annota lo screenshot

Usa la barra degli strumenti sotto al canvas per evidenziare esattamente dove si trova il problema.

  • 1✏ Penna — disegno libero a mano, tiene il tratto fluido
  • 2→ Freccia — disegna una freccia da punto A a punto B
  • 3▭ Rect — rettangolo di selezione (click e trascina)
  • 4T Testo — appare un prompt, scrivi il testo e premi OK
  • 5Scegli uno dei 5 colori e regola lo spessore con lo slider Dim
  • ✓↶ Undo rimuove l'ultima annotazione aggiunta
💡
Il pattern più leggibile: rettangolo rosso attorno all'elemento problematico + freccia rossa che lo indica. Chiarissimo in qualsiasi report.

Passo 5 di 6
Compila il modulo

Il pannello di destra raccoglie tutti i dati strutturati della segnalazione. Compila almeno il campo Problema.

  • 1Tipo: BUG 🐛 per malfunzionamenti, CHANGE REQUEST 🔧 per miglioramenti
  • 2ID: generato automaticamente (BUG-01, MOD-01…) — puoi modificarlo
  • 3Severità: ALTA 🔴 / MEDIA 🟠 / BASSA 🟢
  • 4Problema / Richiesta: descrivi il comportamento errato o la feature richiesta
  • 5Soluzione Proposta: suggerisci come correggere (opzionale)
  • ✓L'Anteprima Output mostra in tempo reale il testo che finirà nel report
🧪
Dettatura vocale (sperimentale): ogni campo ha il proprio pulsante 🎤 Dettatura. Clicca per iniziare, parla, clicca Stop per fermare. Il testo si aggiunge a quello già presente nel campo. La funzione può dipendere da permessi microfono, configurazione di Chrome e disponibilità dell'API vocale.

Passo 6 di 6
Esporta il report

In fondo al pannello destro trovi i due pulsanti di export. Prima di esportare, i dati tecnici (log e network) sono consultabili espandendo le sezioni.

  • 1📄 Salva Report HTML — genera un report completo; Chrome apre la finestra "Salva con nome"
  • 2📷 PNG — salva solo lo screenshot attivo (tab selezionata) con le annotazioni
  • ✓Prima dell'export HTML appare: "Il report può contenere dati sensibili. Continuare?"
  • ✓Il nome file ha il formato: bugforge-report-YYYY-MM-DD…html
⚠️
Almeno uno screenshot obbligatorio: se il canvas è vuoto, il pulsante HTML mostra un errore e blocca l'export. Cattura prima almeno uno screenshot.

Il report HTML

Il report HTML è il formato principale: un file autonomo che contiene tutto il necessario per capire e riprodurre il problema.

📄
Contenuto del report HTML
✓ Titolo pagina e URL sorgente
✓ Data/ora, viewport, User Agent
✓ Tutti gli screenshot con annotazioni
✓ ID, tipo, severità, problema e soluzione
✓ Tabella console logs (livello + timestamp)
✓ Tabella network requests (status, metodo, URL, durata)
📷
Export PNG
✓ Esporta lo screenshot sulla tab attiva
✓ Include tutte le annotazioni disegnate
✓ Ideale per allegare rapidamente a chat/email
✓ Nome automatico con timestamp
📁
Sicurezza dei dati: il content script oscura automaticamente token, JWT e API key presenti negli URL di rete ([REDACTED]). I dati non vengono mai inviati a server esterni — tutto rimane nel tuo browser.

FAQ

Le domande più comuni.

Se BugForge era già aperto viene semplicemente portato in primo piano (non si duplica). Se la finestra appare su un monitor sbagliato, trascinala in posizione: Chrome la ricorderà per le sessioni successive. Se non si apre affatto, ricarica la pagina sorgente e riprova.
Clicca il lucchetto 🔒 nella barra degli indirizzi della finestra BugForge → "Impostazioni sito" → Microfono → "Consenti". Su Windows controlla anche Impostazioni → Privacy → Microfono e assicurati che Chrome abbia il permesso di sistema. BugForge mostra messaggi di errore specifici: "permesso microfono negato" o "microfono non disponibile".
BugForge cattura sempre la finestra sorgente — quella che era attiva quando hai cliccato l'icona per la prima volta. Se hai cambiato tab o aperto nuove finestre nel frattempo, la sorgente rimane quella originale. Per cambiare pagina sorgente chiudi BugForge, vai sulla nuova pagina e riaprilo da lì.
Il collector viene iniettato nella pagina al momento dell'apertura di BugForge. I log prodotti prima non sono recuperabili. Per catturare tutto, apri BugForge prima di eseguire le azioni da monitorare, oppure ricarica la pagina dopo aver aperto BugForge (il collector si reinietta automaticamente alla cattura successiva).
Le pagine di sistema (chrome://settings, chrome://extensions, chrome://newtab, Chrome Web Store) non permettono l'esecuzione di estensioni per motivi di sicurezza del browser. Su qualsiasi altro sito web BugForge funziona normalmente.
No. BugForge non ha backend e non trasmette nulla. Screenshot, log, annotazioni e dati di sistema rimangono nel browser. Eccezione: la funzione di dettatura vocale è sperimentale e usa l'API Web Speech di Chrome, che può comportare elaborazione tramite i servizi vocali del browser. Token e credenziali negli URL di rete vengono oscurati automaticamente con [REDACTED].
BugForge gestisce una segnalazione strutturata per volta (i campi form sono unici). Per più segnalazioni il flusso consigliato è: esporta il report HTML della prima → svuota i campi testo → cattura nuovi screenshot per la seconda. In alternativa puoi aprire più istanze di BugForge da tab diverse del browser.
