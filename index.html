BugForge
Chrome Extension · v1.1.0

Privacy Policy

Effective: June 2025 Last updated: June 2025 Microphone usage declared

01 · Overview

BugForge is a Chrome extension developed by Kaevlar that enables developers and QA professionals to create structured bug reports and change requests directly from the browser. It captures screenshots, annotates them, and collects contextual page data to produce exportable reports.

This Privacy Policy explains what data the extension accesses, how it is used, and what protections are in place. BugForge is designed with a local-first, zero-telemetry philosophy: no data is ever transmitted to external servers operated by the developer.

02 · Data Collected

Data Type Purpose & Details
Page URL & title Included in the bug report to identify the page where the issue was observed. Stored only in the local export file.
Screenshot (visible tab) Captured on explicit user action to document the visual state of the page. Never uploaded or transmitted.
Console logs Up to 200 recent browser console entries (log, warn, error, info, debug) are collected in-memory to assist debugging. Sensitive query parameters (tokens, API keys, JWTs) are automatically redacted before inclusion in any report.
Network requests URL path (excluding query strings), HTTP method, status code, and duration. Full URLs are sanitized — only origin + pathname are retained. No request or response bodies are captured.
User Agent & viewport Browser version and screen dimensions included in the report for reproducibility context.
Voice / microphone audio Used optionally for speech-to-text dictation into report fields. Audio is processed entirely by the browser's built-in Web Speech API (powered by Google's speech recognition service). No audio is stored by the extension or transmitted to developer servers. See Section 04.
Extension preferences Minimal session state (active tab ID, window ID) stored via chrome.storage.session. Cleared automatically when the browser session ends.

03 · Data Not Collected

BugForge does not collect, store, or transmit:

Personal identifiers (name, email, account data) · Passwords or authentication credentials · Payment information · Browsing history across sessions · Cookies or local storage of visited pages · Data from pages the extension is not actively used on · Any data sent to external analytics, advertising, or tracking services.

04 · Microphone & Speech Recognition

BugForge includes an optional voice dictation feature that allows users to dictate text into the "Problem" and "Solution" fields of a bug report using the browser's native Web Speech API (SpeechRecognition).

Important: When voice dictation is used, audio is transmitted by the browser to Google's speech recognition servers for processing. This transmission is handled entirely by Chrome and is subject to Google's Privacy Policy. BugForge does not receive, store, or have access to the audio stream or the raw transcription data — only the final text result inserted into the report field.

Microphone access is requested only when the user explicitly clicks the dictation button. The user may deny microphone permission at any time via Chrome's site settings without affecting any other functionality of the extension.

05 · Data Sharing & Third Parties

BugForge does not share any data with third parties. The extension operates entirely within the user's browser. All report data (screenshots, logs, form fields) is written to a local file on the user's device via the chrome.downloads API and is never transmitted to any server controlled by the developer.

The sole external data transmission that may occur is audio data sent by the browser to Google's speech recognition service, as described in Section 04, and only when the user explicitly activates the dictation feature.

06 · Chrome Permissions Used

Permission Justification
activeTab Captures a screenshot of the currently active tab when the user initiates a report. Access is granted only for the specific tab at the moment of user interaction.
scripting Injects a content script into the active tab to collect console logs and network activity for inclusion in the bug report, on demand.
downloads Saves the completed report (HTML or PNG) to the user's local filesystem via a browser download prompt.
storage Stores temporary session state (active tab and window reference) to coordinate between the extension popup and the source page. No persistent personal data is stored.

07 · Data Retention

BugForge holds data exclusively in-memory during an active reporting session. Session storage (chrome.storage.session) is cleared automatically when the browser session ends. No data persists beyond the user's active session unless explicitly exported to a local file by the user.

Exported report files (HTML/PNG) are stored on the user's own device and are fully under the user's control. The developer has no access to these files.

08 · Remote Code Policy

BugForge does not use any remotely hosted code. All JavaScript, HTML, and CSS assets are bundled within the extension package. No external scripts, CDN resources, or dynamically fetched code are loaded at runtime.

09 · Children's Privacy

BugForge is a professional developer tool not directed at children under the age of 13. The extension does not knowingly collect any personal information from minors.
